2. Understanding CMMC 2.0 Level 1

CMMC 2.0 Level 1, also known as "Foundational," requires organizations to implement 17 basic cybersecurity practices. These practices are designed to protect Federal Contract Information (FCI) – information not intended for public release that is provided by or generated for the Government under a contract. Level 1 focuses on establishing fundamental cyber hygiene and aligns with the safeguarding requirements specified in FAR 52.204-21.

The 17 practices are categorized into six domains:

  • Access Control (AC): 4 Practices
  • Identification & Authentication (IA): 2 Practices
  • Media Protection (MP): 1 Practice
  • Physical Protection (PE): 4 Practices
  • System and Communications Protection (SC): 2 Practices
  • System and Information Integrity (SI): 4 Practices

Compliance with Level 1 is demonstrated through an annual self-assessment submitted to the Supplier Performance Risk System (SPRS).

3. Detailed Control Status & Action Plan

Access Control (AC) – 4 Practices

Practice: Limit information system access to authorized users. In Progress

NIST 800-171 Control: 3.1.1

Requirement Explanation: Ensure that only identified and approved individuals (and processes or devices acting on their behalf) can log in and access company systems. This involves unique user accounts and formal authorization.

MVL's Current Status: Basic password authentication is in place. However, a formal, centrally managed list of authorized users, their roles, and granted access levels is missing. Access is managed system-by-system.

Detailed Remaining Tasks / Action Plan:

  • Develop and approve a formal Access Control Policy and procedures for user authentication and authorization, especially before granting CUI access.
  • Create and maintain a master list of all authorized users, defining identity and role.
  • Synchronize this user list with all critical systems and applications (M365, Procore, Deltek, etc.).
  • Implement Privileged Identity Management (PIM) for IT users as directed by COO.

Suggestions/Comments/COO Directives: COO directed PIM/PAM for IT. Project managers to approve folder access (tracked in Excel for now). Long-term: ITSM tool and central identity management (Entra ID).

Practice: Limit system access to the types of transactions and functions that authorized users are permitted to execute. Not Implemented

NIST 800-171 Control: 3.1.2

Requirement Explanation: Enforce the Principle of Least Privilege. Users should only have permissions necessary for their job functions, restricting what they can do after authentication.

MVL's Current Status: Major gap. Access control lists defining user permissions within applications are not implemented. Excessive permissions and default admin rights are issues.

Detailed Remaining Tasks / Action Plan:

  • IT team to obtain RBAC matrices from Procore, Deltek, and Microtrack administrators.
  • Develop a comprehensive Role-Based Access Control (RBAC) matrix defining permissions for each job role across all applications.
  • Configure all applications to enforce the RBAC matrix.
  • Audit all user accounts and remove local administrator privileges from standard user workstations.

Suggestions/Comments/COO Directives: COO tasked IT with obtaining RBAC matrices. Leverage Zero Trust features in M365 GCC G5 once the matrix is defined.

Practice: Verify and control/limit connections to and use of external information systems. Implemented

NIST 800-171 Control: 3.1.20

Requirement Explanation: Control how internal systems connect to external networks/systems (e.g., third-party cloud services, vendor portals) and manage personally owned devices used for work. Prevent data exfiltration and ingress of threats.

MVL's Current Status: Marked as completed by COO. Controls via Microsoft Intune MDM/MAM are implemented to enforce security policies on company-owned and personal devices, including data wipe capabilities.

Detailed Remaining Tasks / Action Plan:

  • Ensure Intune policies are fully configured, deployed to all in-scope devices, and actively monitored.
  • Periodically review and update MDM/MAM policies.

Suggestions/Comments/COO Directives: COO marked as complete based on Intune MDM implementation plan.

Practice: Control information posted or processed on publicly accessible systems. In Progress

NIST 800-171 Control: 3.1.22

Requirement Explanation: Implement policies and procedures to prevent sensitive information (CUI) from being posted on publicly accessible systems like websites, social media, or public forums.

MVL's Current Status: Public systems are managed by specific departments, but no formal list of authorized posters exists. Procedures to review content for CUI before posting are not established.

Detailed Remaining Tasks / Action Plan:

  • Formalize and document the authorized list of individuals (Abie Musa, Marketing Manager) approved to post information publicly.
  • Create and implement a mandatory "Preparation / Review / Approval" policy and procedure for all public-facing content.
  • Update employee handbook and security awareness training to forbid unauthorized public posting of company information.
  • Communicate the new policy to relevant department heads (Rana & Joshna as per COO directive).

Suggestions/Comments/COO Directives: COO clearly defined authorized posters. Formal policy creation is the key remaining step.

Identification & Authentication (IA) – 2 Practices

Practice: Identify information system users, processes, or devices. In Progress

NIST 800-171 Control: 3.5.1

Requirement Explanation: Assign a unique identifier to every user, device, or automated process accessing systems, forming the basis for accountability.

MVL's Current Status: All employees have unique user IDs via HR onboarding. However, comprehensive documentation of all identities (including service accounts) is incomplete.

Detailed Remaining Tasks / Action Plan:

  • Engage external freelancer to identify and document all system users, service accounts (with owners), and process authentication mechanisms across Microsoft, Zscaler, and Acronis.
  • Ensure all identities are managed through a central system like Microsoft Entra ID.

Suggestions/Comments/COO Directives: COO approved hiring an external freelancer for documentation, highlighting the need for formal records.

Practice: Authenticate (or verify) the identities of those users, processes, or devices. In Progress

NIST 800-171 Control: 3.5.2

Requirement Explanation: Verify the identity presented (e.g., username) typically through passwords, tokens, or biometrics before granting access.

MVL's Current Status: Basic authentication with unique user IDs and passwords is in place. MFA is implemented for email and Teams. However, formal, documented processes for the entire identity lifecycle (creation, password resets, disabling accounts) are missing.

Detailed Remaining Tasks / Action Plan:

  • Formally document procedures for account creation, provisioning, password management (complexity, history, temporary passwords), and disabling accounts upon termination or inactivity.
  • Integrate these processes with the future ITSM tool and HR workflows.
  • Expand MFA to all network access to non-privileged accounts and local/network access to privileged accounts (as per NIST 800-171 full scope).

Suggestions/Comments/COO Directives: Ongoing task linked to ITSM tool implementation and HR process integration. Focus on formalizing the identity lifecycle.

Media Protection (MP) – 1 Practice

Practice: Sanitize or destroy system media containing CUI before disposal or release for reuse. In Progress

NIST 800-171 Control: 3.8.3

Requirement Explanation: Ensure CUI on physical or digital media (hard drives, USBs, tapes, paper) is unrecoverable before disposal or reuse, using methods like shredding, degaussing, or cryptographic erasure.

MVL's Current Status: A paper shredder is available. However, formal, established procedures for sanitizing digital media are not consistently applied or documented.

Detailed Remaining Tasks / Action Plan:

  • Develop and approve a formal Media Protection & Disposal Policy defining approved sanitization/destruction methods (e.g., NIST SP 800-88 guidelines).
  • Implement and document procedures for using Intune's "Data Wipe" for mobile devices/laptops and disk wiping software for loose hard drives.
  • Create and maintain a media disposal log detailing date, method, and personnel for all disposed media.

Suggestions/Comments/COO Directives: COO directed creation of a media disposal log. Technical solution to rely on Intune/Defender for digital media, but formal policy and log are key.

Physical Protection (PE) – 4 Practices

Practice: Limit physical access to systems, equipment, and the respective operating environments to authorized individuals. Not Implemented

NIST 800-171 Control: 3.10.1

Requirement Explanation: Secure physical locations to prevent unauthorized physical access to servers, networking equipment, and workstations, typically via locked doors and access control systems.

MVL's Current Status: Critical deficiency. Assessment could not demonstrate physical access controls. LSA site construction is underway. Biometric access in Dubai office is installed but not used.

Detailed Remaining Tasks / Action Plan:

  • Procure a compliant (non-Chinese, UNIFI-compatible) biometric access control system.
  • Install and activate the new system for all sensitive areas (server rooms, office entrances).
  • Develop and approve a Physical Access Policy defining authorized access to specific areas.
  • Formally designate and document "secure areas" containing CUI systems.

Suggestions/Comments/COO Directives: COO directed purchase of new compliant biometric device. This is a blocking item for several PE controls.

Practice: Escort visitors and monitor visitor activity. Not Implemented

NIST 800-171 Control: 3.10.3

Requirement Explanation: All visitors in facilities must be escorted by an authorized employee, especially near CUI areas, and their activity monitored.

MVL's Current Status: Visitors reportedly escorted, but no robust, formal procedure exists. Practice is informal and not policy-enforced.

Detailed Remaining Tasks / Action Plan:

  • Develop and approve a formal Visitor Management Policy including sign-in/out, badges, and mandatory escorting.
  • Train all employees on the visitor policy and their escorting responsibilities.

Suggestions/Comments/COO Directives: COO directed creation of a formal Visitor Management policy.

Practice: Maintain audit logs of physical access. Not Implemented

NIST 800-171 Control: 3.10.4

Requirement Explanation: Keep records of who enters/leaves secure facilities, via visitor logs and electronic logs from access control systems.

MVL's Current Status: No formal audit logs of physical access. Front desk may manage guests, but no system logs employee access to sensitive areas.

Detailed Remaining Tasks / Action Plan:

  • Implement and configure the new biometric/access control system to log all access events.
  • Ensure logs are retained for a defined period (e.g., 90+ days).
  • Implement a formal visitor sign-in sheet (physical or digital).

Suggestions/Comments/COO Directives: Dependent on new biometric system (3.10.1) implementation for employee logs.

Practice: Control and manage physical access devices. Not Implemented

NIST 800-171 Control: 3.10.5

Requirement Explanation: Properly manage devices controlling physical access (keys, locks, key cards), including issuance, revocation, and preventing compromise.

MVL's Current Status: Physical access controls are yet to be established, so devices are not properly managed.

Detailed Remaining Tasks / Action Plan:

  • Once new biometric system is installed, create an inventory of all physical access credentials.
  • Develop formal procedures for issuing new access credentials and immediately revoking access upon termination or loss.

Suggestions/Comments/COO Directives: Dependent on new biometric system (3.10.1). Procedures for user enrollment/removal in the new system are needed.

System and Communications Protection (SC) – 2 Practices

Practice: Monitor, control, and protect communications at the external boundaries and key internal boundaries of organizational systems. In Progress

NIST 800-171 Control: 3.13.1

Requirement Explanation: Manage information flow at the network perimeter using firewalls, routers, proxies to inspect/filter traffic, blocking malicious/unauthorized communications.

MVL's Current Status: Security controls like firewalls and gateways are not properly in place to segregate networks. Cloud solution communications are not adequately monitored.

Detailed Remaining Tasks / Action Plan:

  • Implement and configure a next-generation firewall at the network perimeter.
  • Define and enforce firewall rules (deny-all by default, permit by exception).
  • Actively monitor inbound/outbound traffic using firewall and Zscaler for threats/anomalies.
  • Establish and conduct the formal six-month Security Architecture Review.

Suggestions/Comments/COO Directives: COO mandated a six-month Security Architecture Review. This recurring review is key.

Practice: Implement subnetworks for publicly accessible system components that are physically or logically separated from internal networks. Implemented

NIST 800-171 Control: 3.13.5

Requirement Explanation: Isolate internet-accessible systems (e.g., public web server) from the internal corporate network, typically in a DMZ protected by firewalls.

MVL's Current Status: Completed. VLANs have been implemented for logical network segmentation.

Detailed Remaining Tasks / Action Plan:

  • Periodically review VLAN configuration to ensure continued security and proper isolation.

Suggestions/Comments/COO Directives: N/A. Control is met.

System and Information Integrity (SI) – 4 Practices

Practice: Identify, report, and correct information and information system flaws in a timely manner. Not Implemented

NIST 800-171 Control: 3.14.1

Requirement Explanation: Implement a vulnerability management process: regularly scan for flaws, have a system for reporting, and a process to fix identified flaws based on severity.

MVL's Current Status: Limited manual analysis for flaws. No formal process for reporting or correcting vulnerabilities.

Detailed Remaining Tasks / Action Plan:

  • Implement automated vulnerability scanning tools.
  • Develop and approve a Patch Management Policy defining remediation timelines based on risk.
  • Utilize the future ITSM tool to track flaws from discovery to remediation.

Suggestions/Comments/COO Directives: COO acknowledged ITSM tool is required for this. This is a significant gap.

Practice: Provide protection from malicious code at designated locations within organizational systems. In Progress

NIST 800-171 Control: 3.14.2

Requirement Explanation: Deploy antivirus/anti-malware on all workstations, servers, and potentially at network gateways (email/web filters).

MVL's Current Status: Endpoint AV (ESET) is used. Protection at network entry/exit points (especially email) is not fully implemented.

Detailed Remaining Tasks / Action Plan:

  • Implement an advanced email security solution to scan/block malicious attachments/links.
  • Utilize firewall and Zscaler capabilities to block known malicious sites and scan web downloads.
  • Granularly retune Microsoft Defender email security settings.

Suggestions/Comments/COO Directives: Ongoing task. COO highlighted need for better email filtering via Defender tuning and potentially a dedicated solution.

Practice: Update malicious code protection mechanisms when new releases are available. Implemented

NIST 800-171 Control: 3.14.4

Requirement Explanation: Ensure anti-malware software is constantly updated with the latest virus signatures and engine updates.

MVL's Current Status: AV signatures are updated daily per pre-defined settings.

Detailed Remaining Tasks / Action Plan:

  • Use the central AV console (Defender) to monitor and verify update success on all endpoints.
  • Conduct periodic manual audits of definition dates on a sample of machines.

Suggestions/Comments/COO Directives: Control is met. Requesting vendor pen-test results is part of broader vulnerability management.

Practice: Perform periodic scans of organizational systems and real-time scans of files from external sources as files are downloaded, opened, or executed. Implemented

NIST 800-171 Control: 3.14.5

Requirement Explanation: Configure anti-malware for regular full-system scans and real-time scanning of new files as they are accessed.

MVL's Current Status: Existing endpoint protection (ESET) performs periodic, real-time, and on-demand scans.

Detailed Remaining Tasks / Action Plan:

  • Transition to and fully utilize Microsoft Defender for Endpoint for enhanced capabilities (behavioral detection, EDR).

Suggestions/Comments/COO Directives: Basic requirement met. Management expects continuous improvement via advanced tools like Defender.
